# 字幕翻訳オーバーレイ設計書

## 実装ステータス
> **✅ 実装完了** (2026-03-10)
> YouTube 字幕の取得強化、Gemini 翻訳、構造化出力、ステータス UI の小型化まで実装済みです。

## 1. 概要

### 1.1 目的

YouTube の文字起こしを動画上へオーバーレイ表示しつつ、Gemini を使った逐次翻訳を重ねて表示できるようにします。字幕取得は YouTube 側の UI や API 変化に左右されやすいため、複数の取得経路を持たせて復元力を高めます。

### 1.2 スコープ

- YouTube 動画ページでの字幕オーバーレイ表示
- 5 行単位の固定集約表示
- 5 ブロック単位の翻訳ウィンドウ
- Gemini への翻訳リクエスト
- 翻訳中は原文をフォールバック表示
- ステータス表示の小型化とエラー可視化

### 1.3 設計原則

- `KISS`: 表示単位と翻訳単位をどちらも 5 に揃え、挙動を単純化しています。
- `YAGNI`: 対象言語は日本語固定、翻訳ジョブは逐次実行のみ、複数モデル切替 UI 以上の機能は追加していません。
- `DRY`: 字幕の正規化、翻訳ウィンドウ管理、ステータス描画は共通関数に寄せています。

## 2. アーキテクチャ

### 2.1 ファイル構成

- `content.js`
  - YouTube DOM 上での字幕取得
  - オーバーレイ UI 描画
  - 再生同期
  - 翻訳キュー管理
- `background.js`
  - Gemini API 呼び出し
  - structured output の schema 指定
  - リトライ制御
- `page-bridge.js`
  - ページコンテキストから `playerResponse` を取得
- `manifest.json`
  - MV3 service worker と API host permission の定義

### 2.2 データフロー

1. `content.js` が現在の動画 ID を検出します。
2. 字幕取得を以下の優先順で試します。
   - Transcript panel の DOM
   - `youtubei/v1/get_transcript`
   - `playerResponse.captionTracks`
   - `ANDROID` / `IOS` client の `youtubei/v1/player`
3. 取得した字幕行を 5 行単位で `groupedSegments` に変換します。
4. `groupedSegments` を 5 ブロック単位の翻訳ウィンドウに切ります。
5. `background.js` が Gemini へ翻訳リクエストを送り、JSON 配列で翻訳結果を返します。
6. `content.js` が index と件数を検証し、`translatedText` として反映します。
7. 再生位置に応じて `translatedText ?? text` を動画上へ表示します。

## 3. 字幕取得設計

### 3.1 多段フォールバック

YouTube の字幕取得は動画ごとに挙動が異なるため、単一経路に依存しません。

- Transcript panel が開けて行が取得できる場合は DOM を使います。
- `get_transcript` が有効な場合は API 応答を使います。
- `WEB` client の `captionTracks` が空本文になる動画があるため、必要時は `ANDROID` と `IOS` の `player` endpoint から字幕トラックを取り直します。

### 3.2 採用理由

- DOM はもっとも人間の見える結果に近いです。
- API は速いですが壊れやすいため補助経路に置いています。
- モバイル client fallback は、`WEB` では空でも字幕本文を返す動画があるため採用しています。

## 4. 翻訳設計

### 4.1 翻訳単位

- 表示単位: 5 行
- 翻訳単位: 5 ブロック

翻訳単位を 20 から 5 に縮めた理由は以下です。

- 初回レスポンスを速くするため
- モデルが複数字幕を結合しやすい問題の影響範囲を小さくするため
- シーク時の追従性を上げるため

### 4.2 実行方式

- 並列ではなく逐次実行
- 現在再生位置の未翻訳ウィンドウを優先
- 翻訳失敗時も原文表示は継続

### 4.3 Structured Output

Gemini には SRT 文字列ではなく、schema 付き JSON 配列を返させます。

返却形:

```json
[
  { "index": 1, "text": "..." },
  { "index": 2, "text": "..." }
]
```

意図:

- SRT 区切り崩れによる件数ズレを減らす
- index 検証で並び替えや欠落を検知する
- timestamp は元データをそのまま使い、モデルには翻訳文だけを担当させる

### 4.4 リトライ方針

- 最大 5 回リトライ
- 一時的失敗だけでなく構造化応答の揺れも再試行対象
- 最終失敗時は対象ウィンドウを `error` にし、他のウィンドウ処理は継続

## 5. UI 設計

### 5.1 字幕オーバーレイ

- 動画上に中央下寄せで表示
- ドラッグ移動可
- ダブルクリックで位置リセット

### 5.2 ステータス表示

- 通常は左上の小さいアイコンだけ表示
- 読み込み開始・読み込み完了・エラー時のみ自動展開
- hover で詳細文を表示
- エラー時は赤アイコン
- 本文は選択してコピー可能
- アイコン自体はドラッグ移動可能

## 6. エラー処理

### 6.1 字幕取得

- 各取得経路の失敗理由を連結して表示
- 最後の一手まで失敗したときだけ致命扱い

### 6.2 翻訳

- JSON 件数不一致
- index 不一致
- 空行過多
- Gemini API エラー

上記はすべてウィンドウ単位の失敗として扱い、アプリ全体は止めません。
