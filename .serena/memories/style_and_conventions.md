# Style and conventions
- User-facing communication for this repo should be in Japanese.
- Follow AGENTS.md: mention design principles explicitly when they are applied (`YAGNI`, `DRY`, `KISS`).
- Read files fully when using shell reads.
- Prefer Bun tooling if package management is introduced; fallback to npm.
- Keep implementation simple and localized in existing files when possible; this repo currently uses plain JavaScript without a build step.
